JWT ( JSON Web Token )
	JSON 포맷을 사용하여 사용자에 대한 정보를 안전하게 전달하기 위한 토큰 기반 인증 방식
	인증된 사용자에 대한 정보를 서버가 서명된 토큰 형태로 발급하고,
	이후 요청 시 클라이언트가 해당 토큰을 보내어 인증을 수행

JWT는 . (점) 으로 구분된 세 부분으로 구성된다
	Header.Payload.Signature
	Header - 토큰 타입과 서명 알고리즘
	Payload - 사용자 정보 및 클레임(Claims)이 담긴 데이터
	Signature - Header와 Payload를 비밀 키로 서명한 값 (위변조 방지)

JWT의 장점
	Stateless - 서버는 인증 상태를 저장하지 않아도 됨
	확장성 - 분산 환경이나 마이크로서비스에 적합
	토큰 자체에 정보 포함 - 별도 조회 없이 인증 가능
	유연한 사용 - 다양한 플랫폼에서 활용 가능 (Web, App, API)

JWT의 단점 및 주의점
	토큰 탈취 시 위험 - 토큰이 클라이언트에 저장되므로 유출 시 치명적
	토큰 만료 전까지 무효화 어려움 - 서버에서 바로 로그아웃 처리 어려움 (블랙리스트 필요)
	Payload는 인코딩이지 암호화가 아님 - 민감 정보는 넣지 말 것
